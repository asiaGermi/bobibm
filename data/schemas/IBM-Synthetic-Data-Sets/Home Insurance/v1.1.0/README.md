### This version introduces the following updates to the datasets. The table lists the dataset name, the columns added or modified, and the corresponding changes:

| Dataset Name   | Column Name                    | Reason                                                                                                                                                                                                                                                                                    |
|:---------------|:-------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| insur_policy   | insurance_Policy_ID            | Previously, the indexes were simple numeric values like 1, 2, 3, which did not appear realistic. A new column has now been added to uniquely identify insurance policies using alphanumeric values, making the dataset more realistic.                                                    |
|                | insurance_Agency_ID            | Previously, the indexes were simple numeric values like 1, 2, 3, which did not appear realistic. A new column has now been added to uniquely identify insurance agencies using alphanumeric values, making the dataset more realistic.                                                    |
|                | Insurance_Agent_Index          | Renaming to improve the readabilty of the dataset                                                                                                                                                                                                                                         |
|                | Insurance_Company_ID           | Renaming to improve the readabilty of the dataset                                                                                                                                                                                                                                         |
| insur_claims   | Insurance_Policy_ID            | Renaming, because earlier this column did not have any refrential integrity with other datasets, now refrential integrity is supported                                                                                                                                                    |
|                | Insurance_Claim_ID             | Adding this column to uniquely identify the insurance claims                                                                                                                                                                                                                              |
|                | Storm_Event_Index              | Replacing the old column with new column, because earlier this column did not have any refrential integrity with storms.csv, now refrential integrity is supported, 0 values in insur_claims.csv for “Storm_Event_Index” means the claim is not associated with a storm                   |
|                | Quake_Event_Index              | Replacing the old column with new column, because earlier this column did not have any refrential integrity with quakes.csv, now refrential integrity is supported, 0 values in insur_claims.csv for “Quake_Event_Index” means the claim is not associated with a earth quake             |
|                | Eruption_Event_Index           | Replacing the old column with new column, because earlier this column did not have any refrential integrity with volcanoes.csv, now refrential integrity is supported, 0 values in insur_claims.csv for “Eruption_Event_Index” means the claim is not associated with a volcanic eruption |
| insur_freetext | Index_to_Ins_Applic_and_Policy | Earlier, freetext column did not hold any context on the which applicant and claim was the freetext generated, this column now tells this information and hence supports refrential integrity                                                                                             |
|                | Insurance_Policy_ID            | This column now tells which policy is the freetext assosciated                                                                                                                                                                                                                            |
|                | Insurance_Claim_ID             | This column now tells which claim is the freetext assosciated                                                                                                                                                                                                                             |
| All            | All                            | In the previous datasets, there were spaces in between the column names of the datasets, these spaces caused issues when the datasets were uploaded to databases like Db2. This issues has been resolved by adding underscores in column names.                                           |
